## Codebase Patterns

### Product goal (read first)
- Clicky is now being built as a shippable personal tool for end users ŌĆö not a dev playground. Every user installs the `.exe`, enters their own API keys in a first-run settings window, and runs the app against their own quota on their own machine. There is no shared backend. The `worker/` directory exists for the Mac reference implementation and is unused by the Windows client.
- Keys are encrypted at rest using Windows DPAPI (`ProtectedData.Protect` with `DataProtectionScope.CurrentUser`) and stored at `%APPDATA%\Clicky\secrets.bin`. Non-secret settings live at `%APPDATA%\Clicky\settings.json`.
- UI copy is user-facing, not developer-facing. Prefer "Couldn't reach Anthropic ŌĆö check your internet connection or your API key in Settings" over "HTTP 401 Unauthorized". Avoid words like "provider", "endpoint", "SSE", "proxy".
- Target vision model for GLM: `glm-4.6v`. GLM-5.1, GLM-5, GLM-5-turbo, GLM-4.5-air are **text-only** and must not be offered ŌĆö they silently break the `[POINT:x,y:label:screenN]` screen-pointing feature.

### Repo layout and branches
- Monorepo: `mac/` is the Swift/Xcode reference, `windows/` is the C# / .NET 8 / WPF port, `worker/` is legacy shared Cloudflare proxy (used by Mac only).
- Branch: stay on `ralph/windows-port`. The completed Windows port lives there and is NOT on main yet. Creating a new branch from main would lose it.
- Do not edit `mac/` or `worker/` ŌĆö this PRD is client-side only.

### Architecture after GLM integration (target state)
- `Clicky.Api.ILlmClient` is the common LLM interface: `IAsyncEnumerable<string> SendAsync(history, screens, systemPrompt, userText, ct)`. Two implementations: `AnthropicDirectClient` (ŌåÆ api.anthropic.com) and `ZaiDirectClient` (ŌåÆ api.z.ai OpenAI-compat). Both live in `Clicky.Api` and take their API key as a constructor parameter.
- `AssemblyAiStreamingTranscriber` fetches its session token directly from `https://streaming.assemblyai.com/v3/token` with the raw user key in the `Authorization` header (NOT Bearer ŌĆö AssemblyAI's token endpoint expects the bare key).
- `ElevenLabsTtsClient` POSTs directly to `https://api.elevenlabs.io/v1/text-to-speech/{voiceId}/stream` with `xi-api-key: <userKey>`.
- `Clicky.Companion.SecretsStore` + `SettingsStore` are the single source of truth for user config. `App.xaml.cs` reads them on startup and constructs all four clients with the values.
- `Clicky.Companion.CompanionManager` takes `ILlmClient`, `AssemblyAiStreamingTranscriber`, `ElevenLabsTtsClient` via constructor injection. It must NOT know which concrete LLM implementation it's holding ŌĆö that knowledge lives in App.xaml.cs's factory.
- `Clicky.App.SettingsWindow` is reachable from the tray right-click menu's `Settings...` item, auto-opens on first launch, and auto-reopens if required keys are missing or invalid at runtime.
- Runtime model switching happens via a tray `Model` submenu that calls `CompanionManager.SwapLlmClientAsync(provider, model)`.

### GLM / OpenAI-compat translation rules
- z.ai OpenAI-compat endpoint: `https://api.z.ai/api/paas/v4/chat/completions`. Auth: `Authorization: Bearer <ZAI_API_KEY>`.
- Request translation (Anthropic ŌåÆ OpenAI chat-completions): `system` prompt ŌåÆ `messages[0] = {role:"system", content: systemPrompt}`; text content blocks ŌåÆ `{type:"text", text}`; image content blocks ŌåÆ `{type:"image_url", image_url:{url:"data:<media_type>;base64,<base64>"}}`; `max_tokens`/`temperature`/`stream:true` preserved.
- Response translation (OpenAI SSE ŌåÆ string deltas): read `data: {...}` lines, extract `choices[0].delta.content`, yield as strings. `[DONE]` sentinel ends the stream. Skip malformed JSON lines, don't throw. Use an incremental UTF-8 decoder so multi-byte chars split across HTTP chunks don't corrupt.
- System prompt (built in CompanionManager from US-010, includes `[POINT:x,y:label:screenN]` instructions) passes through unchanged ŌĆö GLM-4.6V is prompted in the same English, it just speaks OpenAI over the wire.

### DPAPI gotchas
- DPAPI scope is per-user-account. A backup restored to a different Windows user cannot decrypt. This is correct security behavior ŌĆö do not add a passphrase fallback.
- DPAPI throws `CryptographicException` on decryption failure ŌĆö catch it and return null from `SecretsStore.Read`, then let the caller (App.xaml.cs startup) treat it as "missing key" and open SettingsWindow.
- Secrets file format: encrypt the whole JSON dictionary as one blob, not per-entry. Single write on every change.
- `System.Security.Cryptography.ProtectedData` requires `<PackageReference Include="System.Security.Cryptography.ProtectedData" Version="..." />` ŌĆö it's no longer in the default .NET 8 surface for non-Windows, but Clicky targets `net8.0-windows10.0.19041.0` so it's available; still add the explicit package reference to be safe.

### Test API endpoints (for SettingsWindow Test buttons, US-022)
- **Anthropic**: `POST https://api.anthropic.com/v1/messages` with `{"model":"claude-haiku-4-5","max_tokens":1,"messages":[{"role":"user","content":"hi"}]}`. ~$0.0001 per probe. Headers: `x-api-key`, `anthropic-version: 2023-06-01`.
- **z.ai**: `POST https://api.z.ai/api/paas/v4/chat/completions` with `{"model":"glm-4.5v","max_tokens":1,"messages":[{"role":"user","content":"hi"}]}`. Headers: `Authorization: Bearer <key>`.
- **AssemblyAI**: `GET https://streaming.assemblyai.com/v3/token?expires_in_seconds=60`. Headers: `Authorization: <rawKey>` (no Bearer). Returns a JWT on success, 401 on bad key.
- **ElevenLabs**: `GET https://api.elevenlabs.io/v1/user/subscription`. Headers: `xi-api-key: <key>`. Returns subscription info on success, 401 on bad key. **This is the free auth check ŌĆö do not run an actual TTS synthesis just to test the key.**

### C# / .NET gotchas carried from Windows port (full details in archive/progress-windows-port.txt)
- All projects target `net8.0-windows10.0.19041.0` (not plain `net8.0-windows`). Changing one project's TFM cascades to all referencing projects.
- `internal` members need `<InternalsVisibleTo Include="Clicky.Tests" />` in the library .csproj for test access.
- `ReadOnlySpan<T>` cannot be used inside `async` iterator methods (`IAsyncEnumerable`) ŌĆö use `string`/`Substring`.
- Git user config is not set on this Windows host ŌĆö use `git -c user.name="rickkdev" -c user.email="mail@riccardolamanna.de"` for commits.
- `CompanionManager` owns the hotkey consumer loop and all state transitions. `App.xaml.cs` constructs it and calls `Start()` / `Dispose()` ŌĆö does not consume hotkey transitions directly.
- TTS pipeline: `SentenceTokenizer` splits LLM deltas at `.!?\n\n` boundaries ŌåÆ `TtsPipeline` (Channel-based) plays sentences sequentially via `ElevenLabsTtsClient.SpeakAsync`. `ElevenLabsTtsClient` is NOT sealed (virtual `SpeakAsync` for test mocking).
- Capture-on-press: `HandlePressed()` kicks off `ScreenCapture.CaptureAllScreensAsJpegAsync` immediately; result is awaited in `SendTranscriptToClaudeWithScreenshot`. Tap < 100 ms ŌåÆ cancel and re-capture.
- Screenshot labels must include `(image dimensions: WxH pixels)` ŌĆö appended in CompanionManager, not in ScreenCapture.
- Image media_type: sniff PNG vs JPEG by magic bytes. Both Anthropic and z.ai's OpenAI-compat reject mismatched declared types.
- AssemblyAI: reuse a single long-lived `HttpClient` / `ClientWebSocket` factory across sessions. Per-session sockets corrupt the connection pool.
- Push-to-talk hook must be listen-only (always `CallNextHookEx`) ŌĆö never swallow keys.
- Per-monitor DPI: every WPF window must opt into PerMonitorV2 via `app.manifest`. Overlay coordinate math breaks on mixed-DPI setups otherwise.
- WPF `Window.Left/Top/Width/Height` are always DIPs. Win32 APIs (`GetMonitorInfo`, `GetCursorPos`) return physical pixels. Convert via `DpiHelper.ToDips()` at the WPF boundary. `PointTagParser` stays in physical pixels.
- Overlay windows must set `WS_EX_TRANSPARENT | WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE` via `SetWindowLong` in `OnSourceInitialized`.
- WGC Win11+ APIs (IsBorderRequired, IsCursorCaptureEnabled) must be called via reflection when targeting SDK 10.0.19041.
- `LibraryImport` source generators don't handle delegate params with `ref` ŌĆö use `DllImport` for callbacks like `MonitorEnumProc`.

- Desktop pointing smoke verification is tray-driven: right-click tray icon -> `Desktop Smoke Test`, click `box A`/`box B`/`box C`, then inspect `%APPDATA%\Clicky\point-debug\*-raw.jpg`, `*-annotated.png`, and `%APPDATA%\Clicky\debug.log` `[SMOKE]`/`[POINT]` lines. This path is desktop-only and should stay outside default CI.
- Full-turn CI coverage should stay deterministic: use `CompanionManager`'s fake capture/overlay seams with fake LLM and TTS clients; keep real desktop compositor, visible WPF overlay, microphone, speakers, and provider API keys out of default `dotnet test`.
- Synthetic box fixtures prove explicit coordinate conversion and overlay last mile only. They do not prove semantic AI localization. Real pointing work must be validated with real screenshot fixtures and allowed/distractor regions for failures like Egypt->Libya or Algeria->ocean.
- Windows pointing is back to a single-response baseline: one normal LLM request may include one `[POINT]` tag, then `PointTagParser` converts that explicit screenshot coordinate to physical pixels and the overlay dispatches it. Do not re-add locator/verifier/fallback LLM calls until the redesign and real-fixture evaluator prove a new protocol.
- The Windows pointing redesign contract lives in `windows/docs/pointing-redesign.md`. The US-049 structured `{ spokenText, pointIntent }` path is behind `CompanionManager`'s `useRedesignedPointingProtocol` switch and is default-off until US-050 records real desktop evidence. Keep verifier/locator/fallback layers out unless the fixture gate proves a need.
- As of US-050, `Clicky.App.App.UseRedesignedPointingProtocolByDefault` enables the structured pointing path for the packaged Windows app. Keep the `CompanionManager` constructor switch available for deterministic tests and comparison paths.
- Windows turn timing evidence uses `[TURN]` lines in `%APPDATA%\Clicky\debug.log` with a correlation id, provider/model, capture payload bytes, cursor-screen filtering state, LLM/TTS milestones, point parse, and overlay dispatch. Use tray `Provider Timing Diagnostics` for real provider comparisons: inspect `diagnostic-text-llm-*`, `diagnostic-image-llm-*`, `capture-*`, and `diagnostic-tts-*` before blaming overlay rendering or model vision.
- Semantic pointing fixtures live in `windows/tests/Clicky.Tests/Fixtures/pointing/`: add screenshot-style PNGs, allowed polygons, distractor polygons, and canned outputs there. Run `SemanticPointingEvaluationTests` before changing semantic map/UI pointing behavior.

## 2026-04-11 - US-026

Three tasks were discarded with `_ = ...`: `ConsumeHotkeyTransitionsAsync` (line 110), `StartRecordingAsync` (line 160), and `StopRecordingAndProcessAsync` (line 173). - Fix 1: Converted `CompanionManager` from `IDisposable` to `IAsyncDisposable` (also keeps `IDisposable` as synchronous fallback). `DisposeAsync` cancels all CancellationTokenSources, then awaits all tracked background tasks with a 2-second `Task.WhenAll(...).WaitAsync(TimeSpan)` timeout. If tasks don't finish in 2s, cleanup proceeds anyway.
- Fix 2: Stored all previously fire-and-forget tasks as `Task` fields (`_hookConsumerTask`, `_recordingTask`, `_stopRecordingTask`) so `DisposeAsync` can await them. `_currentResponseTask` was already tracked.
- Fix 3: Added `_disposed` guard to both `Dispose` and `DisposeAsync` for idempotency ŌĆö prevents `ObjectDisposedException` on double-dispose.
- Fix 4: `App.OnExit` now calls `_companionManager.DisposeAsync().AsTask().Wait(TimeSpan.FromSeconds(2))` instead of synchronous `Dispose()`, giving background tasks time to wind down. Disposal order is deterministic: (1) timers/analytics, (2) CompanionManager (async, 2s timeout), (3) overlay manager, (4) push-to-talk hook, (5) tray icon.
- Fix 5: Added `Environment.Exit(0)` safety net via `Task.Delay(500).ContinueWith(...)` after `base.OnExit(e)` ŌĆö belt-and-braces to prevent the process lingering if an unforeseen thread survives the dispose chain.
