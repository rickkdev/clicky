## Codebase Patterns

### Product goal (read first)
- Clicky is now being built as a shippable personal tool for end users — not a dev playground. Every user installs the `.exe`, enters their own API keys in a first-run settings window, and runs the app against their own quota on their own machine. There is no shared backend. The `worker/` directory exists for the Mac reference implementation and is unused by the Windows client.
- Keys are encrypted at rest using Windows DPAPI (`ProtectedData.Protect` with `DataProtectionScope.CurrentUser`) and stored at `%APPDATA%\Clicky\secrets.bin`. Non-secret settings live at `%APPDATA%\Clicky\settings.json`.
- UI copy is user-facing, not developer-facing. Prefer "Couldn't reach Anthropic — check your internet connection or your API key in Settings" over "HTTP 401 Unauthorized". Avoid words like "provider", "endpoint", "SSE", "proxy".
- Target vision model for GLM: `glm-4.6v`. GLM-5.1, GLM-5, GLM-5-turbo, GLM-4.5-air are **text-only** and must not be offered — they silently break the `[POINT:x,y:label:screenN]` screen-pointing feature.

### Repo layout and branches
- Monorepo: `mac/` is the Swift/Xcode reference, `windows/` is the C# / .NET 8 / WPF port, `worker/` is legacy shared Cloudflare proxy (used by Mac only).
- Branch: stay on `ralph/windows-port`. The completed Windows port lives there and is NOT on main yet. Creating a new branch from main would lose it.
- Do not edit `mac/` or `worker/` — this PRD is client-side only.

### Architecture after GLM integration (target state)
- `Clicky.Api.ILlmClient` is the common LLM interface: `IAsyncEnumerable<string> SendAsync(history, screens, systemPrompt, userText, ct)`. Two implementations: `AnthropicDirectClient` (→ api.anthropic.com) and `ZaiDirectClient` (→ api.z.ai OpenAI-compat). Both live in `Clicky.Api` and take their API key as a constructor parameter.
- `AssemblyAiStreamingTranscriber` fetches its session token directly from `https://streaming.assemblyai.com/v3/token` with the raw user key in the `Authorization` header (NOT Bearer — AssemblyAI's token endpoint expects the bare key).
- `ElevenLabsTtsClient` POSTs directly to `https://api.elevenlabs.io/v1/text-to-speech/{voiceId}/stream` with `xi-api-key: <userKey>`.
- `Clicky.Companion.SecretsStore` + `SettingsStore` are the single source of truth for user config. `App.xaml.cs` reads them on startup and constructs all four clients with the values.
- `Clicky.Companion.CompanionManager` takes `ILlmClient`, `AssemblyAiStreamingTranscriber`, `ElevenLabsTtsClient` via constructor injection. It must NOT know which concrete LLM implementation it's holding — that knowledge lives in App.xaml.cs's factory.
- `Clicky.App.SettingsWindow` is reachable from the tray right-click menu's `Settings...` item, auto-opens on first launch, and auto-reopens if required keys are missing or invalid at runtime.
- Runtime model switching happens via a tray `Model` submenu that calls `CompanionManager.SwapLlmClientAsync(provider, model)`.

### GLM / OpenAI-compat translation rules
- z.ai OpenAI-compat endpoint: `https://api.z.ai/api/paas/v4/chat/completions`. Auth: `Authorization: Bearer <ZAI_API_KEY>`.
- Request translation (Anthropic → OpenAI chat-completions): `system` prompt → `messages[0] = {role:"system", content: systemPrompt}`; text content blocks → `{type:"text", text}`; image content blocks → `{type:"image_url", image_url:{url:"data:<media_type>;base64,<base64>"}}`; `max_tokens`/`temperature`/`stream:true` preserved.
- Response translation (OpenAI SSE → string deltas): read `data: {...}` lines, extract `choices[0].delta.content`, yield as strings. `[DONE]` sentinel ends the stream. Skip malformed JSON lines, don't throw. Use an incremental UTF-8 decoder so multi-byte chars split across HTTP chunks don't corrupt.
- System prompt (built in CompanionManager from US-010, includes `[POINT:x,y:label:screenN]` instructions) passes through unchanged — GLM-4.6V is prompted in the same English, it just speaks OpenAI over the wire.

### DPAPI gotchas
- DPAPI scope is per-user-account. A backup restored to a different Windows user cannot decrypt. This is correct security behavior — do not add a passphrase fallback.
- DPAPI throws `CryptographicException` on decryption failure — catch it and return null from `SecretsStore.Read`, then let the caller (App.xaml.cs startup) treat it as "missing key" and open SettingsWindow.
- Secrets file format: encrypt the whole JSON dictionary as one blob, not per-entry. Single write on every change.
- `System.Security.Cryptography.ProtectedData` requires `<PackageReference Include="System.Security.Cryptography.ProtectedData" Version="..." />` — it's no longer in the default .NET 8 surface for non-Windows, but Clicky targets `net8.0-windows10.0.19041.0` so it's available; still add the explicit package reference to be safe.

### Test API endpoints (for SettingsWindow Test buttons, US-022)
- **Anthropic**: `POST https://api.anthropic.com/v1/messages` with `{"model":"claude-haiku-4-5","max_tokens":1,"messages":[{"role":"user","content":"hi"}]}`. ~$0.0001 per probe. Headers: `x-api-key`, `anthropic-version: 2023-06-01`.
- **z.ai**: `POST https://api.z.ai/api/paas/v4/chat/completions` with `{"model":"glm-4.5v","max_tokens":1,"messages":[{"role":"user","content":"hi"}]}`. Headers: `Authorization: Bearer <key>`.
- **AssemblyAI**: `GET https://streaming.assemblyai.com/v3/token?expires_in_seconds=60`. Headers: `Authorization: <rawKey>` (no Bearer). Returns a JWT on success, 401 on bad key.
- **ElevenLabs**: `GET https://api.elevenlabs.io/v1/user/subscription`. Headers: `xi-api-key: <key>`. Returns subscription info on success, 401 on bad key. **This is the free auth check — do not run an actual TTS synthesis just to test the key.**

### C# / .NET gotchas carried from Windows port (full details in archive/progress-windows-port.txt)
- All projects target `net8.0-windows10.0.19041.0` (not plain `net8.0-windows`). Changing one project's TFM cascades to all referencing projects.
- `internal` members need `<InternalsVisibleTo Include="Clicky.Tests" />` in the library .csproj for test access.
- `ReadOnlySpan<T>` cannot be used inside `async` iterator methods (`IAsyncEnumerable`) — use `string`/`Substring`.
- Git user config is not set on this Windows host — use `git -c user.name="rickkdev" -c user.email="mail@riccardolamanna.de"` for commits.
- `CompanionManager` owns the hotkey consumer loop and all state transitions. `App.xaml.cs` constructs it and calls `Start()` / `Dispose()` — does not consume hotkey transitions directly.
- Screenshot labels must include `(image dimensions: WxH pixels)` — appended in CompanionManager, not in ScreenCapture.
- Image media_type: sniff PNG vs JPEG by magic bytes. Both Anthropic and z.ai's OpenAI-compat reject mismatched declared types.
- AssemblyAI: reuse a single long-lived `HttpClient` / `ClientWebSocket` factory across sessions. Per-session sockets corrupt the connection pool.
- Push-to-talk hook must be listen-only (always `CallNextHookEx`) — never swallow keys.
- Per-monitor DPI: every WPF window must opt into PerMonitorV2 via `app.manifest`. Overlay coordinate math breaks on mixed-DPI setups otherwise.
- Overlay windows must set `WS_EX_TRANSPARENT | WS_EX_TOOLWINDOW | WS_EX_NOACTIVATE` via `SetWindowLong` in `OnSourceInitialized`.
- WGC Win11+ APIs (IsBorderRequired, IsCursorCaptureEnabled) must be called via reflection when targeting SDK 10.0.19041.
- `LibraryImport` source generators don't handle delegate params with `ref` — use `DllImport` for callbacks like `MonitorEnumProc`.
